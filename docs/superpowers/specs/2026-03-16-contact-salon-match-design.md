# Contact Salon Match — Design Spec

**Date:** 2026-03-16
**Status:** Draft
**Branch:** feature/contact-salon-match
**Purpose:** Scan user's phone contacts against BeautyCita's discovered/registered salon database. Show matches with "Reservar en BeautyCita" action. On Android, register as a contact action provider so "Book in BeautyCita" appears in the native contacts app.

---

## Core Principle

Your phone already knows your favorite salons — they're in your contacts. BeautyCita finds them and gives you a one-tap booking shortcut. All matching happens on-device. We never upload or see your contacts.

---

## Privacy Model

- Contacts permission requested with clear explanation: "Busca salones en tus contactos para reservar mas rapido"
- Phone numbers downloaded from BeautyCita DB (MX salons only, ~38K numbers)
- Matching runs locally in Dart — no contact data leaves the device
- User can revoke at any time from settings
- LFPDPPP compliant: no personal data transmitted

---

## Level 1: In-App Contact Matching (iOS + Android)

### Entry Point
New section on the home screen below the category grid: "Salones en tus contactos". Shows after contacts permission is granted. If permission not yet requested, shows a CTA card: "Encuentra tus salones favoritos" with a button to request permission.

### Permission Flow
1. User sees CTA card on home screen
2. Taps "Buscar en mis contactos"
3. System permission dialog appears
4. If granted: scan runs, matches displayed
5. If denied: CTA card shows "Permiso necesario" with link to settings
6. Permission status persisted in SharedPreferences

### Matching Engine
1. **Download salon phone list** — edge function returns a compact list of `{phone, salon_id}` pairs for MX salons only. Cached locally for 24 hours.
   - Endpoint: `outreach-discovered-salon` new action `phone_list`
   - Returns: JSON array of `{p: "+5215551234567", id: "uuid", t: "d"}` where t = "d" (discovered) or "r" (registered)
   - ~38K entries, compressed ~400KB

2. **Read device contacts** — use `flutter_contacts` package to read all contacts with phone numbers. Extract and normalize phone numbers (strip spaces, dashes, add +52 prefix if missing).

3. **Match** — for each contact phone number, check against the downloaded salon set (HashMap lookup, O(1) per contact). Collect matches.

4. **Enrich matches** — for each matched salon_id, fetch full salon details (name, photo, rating, categories) from Supabase. Batch query.

5. **Display** — horizontal scroll card row on home screen showing matched salons with the contact name the user saved them as + the salon's BeautyCita data.

### Match Display Card
Each matched salon shows:
- Contact name (from user's phone, e.g., "Mi estilista Lupita")
- Salon name from DB (e.g., "Salon Lupita")
- Photo (from DB or contact photo)
- Rating + review count
- Category chips
- **If registered salon**: "Reservar" button → opens booking flow
- **If discovered salon**: "Invitar a BeautyCita" button → opens invite detail with pre-written Aphrodite message (same engine as invite experience). Message is ALREADY written when they see it — no "generate" step. Only option is "Redo" to get a different version.
- **Contact actions row**: Call, WhatsApp, Book/Invite in BC — three icon buttons

### Discovered Salon Detail (from contact match)
When user taps a discovered salon match:
- "Acerca de este salon" — pre-generated by Aphrodite using all enriched data (name, category, city, rating, IG bio, review count, specialties). Unique and personal per salon. Generated on first view, cached in `generated_bio`.
- Invite message — already written and displayed in the WhatsApp bubble. User reads it, taps "Redo" if they want a different version, then "Enviar" to open WhatsApp with the message pre-filled.
- No "Generate" button — the message exists from the moment they see the screen. Only "Redo" (refresh icon) to get an alternative.

### Refresh
- Auto-scan on app launch if permission granted + cache expired (>24h)
- Pull-to-refresh on home screen triggers rescan
- New matches trigger a subtle notification badge

## Level 2: Android Contact Action Provider (Android only)

Register BeautyCita as a SyncAdapter so that matched contacts show a "Reservar en BeautyCita" action in the native Android Contacts app.

### How Android Contact Actions Work
1. App registers a `SyncAdapter` + `AccountAuthenticator` (even if no actual sync happens)
2. App creates a BeautyCita "account" in Android's Account Manager
3. For each matched contact, app inserts a `RawContact` linked to the existing contact via `AggregationExceptions`
4. The raw contact has a single `Data` row with a custom MIME type (e.g., `vnd.android.cursor.item/com.beautycita.book`)
5. Android shows this as an action row in the contact's detail view: "Book in BeautyCita" with the BC icon

### Implementation
- **Native Kotlin** code in `android/app/src/main/kotlin/`
  - `BeautyCitaAccountAuthenticator.kt` — stub authenticator
  - `BeautyCitaSyncAdapter.kt` — triggered after in-app matching, writes raw contacts
  - `BeautyCitaContactActionActivity.kt` — launched when user taps "Book in BeautyCita" in native contacts
- **Flutter MethodChannel** bridge to trigger sync from Dart
- **AndroidManifest.xml** additions: authenticator service, sync adapter service, contacts permissions

### What the User Sees (in native Android Contacts)
When viewing a contact that matches a BeautyCita salon:
```
Maria's Salon
+52 322 123 4567

  Call          Message        Video

  Book in BeautyCita    ← NEW (BC gradient icon)
```

Tapping "Book in BeautyCita" opens the app directly to the booking flow for that salon (deep link).

### iOS Limitation
iOS does not allow third-party apps to add actions to native contacts. Level 2 is Android-only. On iOS, the in-app matching (Level 1) is the full experience.

## Edge Function: phone_list action

New action in `outreach-discovered-salon`:

```typescript
if (action === "phone_list") {
  // No auth required — this is a public phone number list (no PII beyond phone)
  // Only MX salons
  const { data } = await serviceClient
    .from("discovered_salons")
    .select("id, phone")
    .in("country", ["MX", "Mexico"])
    .not("phone", "is", null)
    .limit(50000);

  // Also include registered businesses
  const { data: registered } = await serviceClient
    .from("businesses")
    .select("id, phone")
    .eq("is_active", true)
    .not("phone", "is", null);

  // Compact format
  const phones = [
    ...(data ?? []).map(s => ({ p: s.phone, id: s.id, t: "d" })),
    ...(registered ?? []).map(b => ({ p: b.phone, id: b.id, t: "r" })),
  ];

  return jsonResponse({ phones, count: phones.length });
}
```

Cache headers: `Cache-Control: public, max-age=86400` (24 hours).

- `contacts_phone_cache_timestamp` (ISO string)
